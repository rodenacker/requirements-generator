# research-reference.md

**Purpose:** Methodology reference for competitor / comparable-application research. Statement form per finding (competitor → capability/pattern → relevance: alignment/divergence/opportunity); sources (consultant-supplied URLs, named comparables, search terms; web search where authorised); quality checks (every cited capability has source URL or screenshot reference; every finding has one-line relevance rationale grounded in `requirements.md > domain` or §User goals; no unsourced claims); stop-condition.

**Used by:**
- `framework/agents/researcher/agent.md` — drives the agent's process.
- `framework/skills/research-web.md` — fetch + summarise pages.
- `framework/skills/synthesise-research-report.md` — assembles findings into the comparative report.

**How used:** Loaded by the researcher agent at activation. Lives at `assets/research-reference.md` (top-level), **not** under `analyse-requirements/` — research is external-evidence gathering, distinct from analyses (UX methodologies applied to requirements).
