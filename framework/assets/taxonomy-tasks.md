# taxonomy-tasks.md

**Purpose:** Defines the five task scopes — multi-screen, screen, multi-view, view, interaction — plus the scope-classification signals (Source UI structure, UX-pattern preference, step & field counts, task verb, data entities touched, frequency × stakes).

**Used by:**
- `framework/agents/requirements-drafter/agent.md` — when classifying §Task flows.
- `framework/agents/design-spec-drafter/agent.md` — task scope drives screen/view/interaction decomposition.
- `framework/skills/derive-ui-inventory.md` — scope → containment shape.

**How used:** Loaded by drafters. Determines whether a task becomes a multi-screen wizard, a one-screen workflow, a tabbed sub-view, or an interaction within an existing element. Wraps the consultant's natural-language task descriptions in v7b's structural vocabulary.
