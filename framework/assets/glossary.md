# glossary.md

**Purpose:** Canonical terms across four layers — **BA layer** (Domain Model, Concept, Aggregate, Ubiquitous Language, Invariant, Lifecycle State); **Define** (User Goal, Journey, Task, Layout, UX Pattern); **UI elements** (Screen, View, Organism, Molecule, Atom + Trade-off Dimension as the evaluation layer); **v7b additions** (revision marker, stage receipt, object map, ORCA, CCP, CTA, taxonomy_id, taxonomy_kind, taxonomy_modifier, delta facts, completeness report, AI-suggested marker). Plus the dropped-term replacements table (page → screen; control → atom/molecule; widget → atom/molecule/organism; element → be specific; feature → n/a; module → BMAD legacy; canvas/frame → Pencil-only). Plus persona disambiguation (Unicorn / Consultant / target-user persona). **BA-layer / OOUX disambiguation:** Concept (BA, in `requirements.md > §2.1`) is the business-language atom — Object (OOUX, in `analyse-requirements/OOUX/ooux-object-map.html`) is its UX-lens refinement carrying CTAs and CCPs.

**Used by:**
- All agents — vocabulary discipline.
- `framework/skills/completeness-report.md` — flags dropped-term usage as a `vague` or `contradictory` finding.
- `framework/agents/requirements-drafter/agent.md`, `design-spec-drafter` — uses canonical terms, never legacy ones.
- All character files — tone-floor references.

**How used:** Loaded as a vocabulary reference. Adding a new term requires updating this file. Dropped-term usage in any spec is a finding.
