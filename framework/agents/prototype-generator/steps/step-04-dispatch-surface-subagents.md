# step-04-dispatch-surface-subagents

**Goal:** Render each surface's new shared components in parallel, safely.

1. Group the render plan by surface. **Dispatch every surface that has `owned_files` in a single wave** — one Agent-tool message with N `Agent` calls — up to a ceiling of **8**. If >8 surfaces have owned components (rare; rule 13 shrinks this over runs), batch in waves of ≤8. The disjoint `owned_files` partition (verified collision-free in step-02), **not** the wave size, is the collision-safety boundary and holds at any N; the harness caps true concurrency at ~min(16, cpu-2) and queues the remainder gracefully. A surface with no `owned_files` at all — no new components **and** not a parallelizable standalone route (i.e. a fold, a wizard, or the primary/root surface whose components are all reuse) — needs no sub-agent; the driver composes it directly in step-05. A **standalone secondary** surface always has at least its own route file in `owned_files` (step-02), so it is always dispatched, even when all its components are reuse.
2. **Timing — open the wave.** Before dispatching, emit `substep_start` (`stage:"generator"`, `substep:"surface-wave"`) per `prototype-generator.md > Timing log (sub-steps)`. Dispatch all sub-agents in a **single Agent-tool message** (one tool block, multiple `Agent` calls — like `wireframe-orch` Stage 3). Each sub-agent prompt:
   - names `framework/agents/prototype-generator/steps/step-sub-render-surface.md` as its instructions;
   - passes the surface assignment: `{ surface_id, surface_intent, realization, host (if folded), owned_files (the ONLY files it may write — for a standalone secondary surface this includes its own route page), owned_route_file (its `src/app/<name_slug>/<surface-kebab>/page.tsx`, or null for non-parallelizable surfaces), route_map (the outbound link target route strings that originate at this surface, from spec §6 flows, + this surface's breadcrumb trail — knowledge only, so the page can wire `<Link href>` to real routes without knowing how the targets are realized), nav_context (whether a driver-authored per-prototype `layout.tsx` supplies shared nav this page inherits, or the page wires its own back-link/breadcrumb), components_to_reuse, data_bindings (Property→store), role_visibility (spec §6.5 visible-components/actions per role for this surface when multi-role; else null — lets a standalone page wire `activeRole` itself), application_character (the compact spec-§3 block: name + tone attributes + per-surface copy guidance, or null), posture, dimension_positions, ux-baseline subset }`;
   - states the hard rule: **write only `owned_files`; read the rest of the library; never write the data layer; never write any route file other than your own `owned_route_file` (standalone surfaces only — never the root `page.tsx`, a per-prototype `layout.tsx`, a folded-host route, or a wizard route); never overwrite an existing file.**
3. Await all. Each returns a **(route) manifest**: `{ surface_id, files_written, components_created, components_reused, props_bound, states_rendered, baseline_ok, started, ended }` — where `started`/`ended` are the ISO timestamps the sub-agent **self-measured** for its own render (timing only; the driver, not the sub-agent, writes them to the log) — plus, for a standalone secondary surface that authored its own route, `route_written, primary_cta_present, outbound_links: [route_path,…]` (or `failed {surface_id, reason}`). step-05 consumes the route fields to skip re-authoring; step-07 self-validates against them.
4. **Timing — close the wave + per-surface spans.** After the wave joins, emit `substep_end` (`substep:"surface-wave"`); then, **serially on the driver thread**, emit one `render-surface` pair per returned manifest — `substep_start`/`substep_end` (`stage:"generator"`, `substep:"render-surface"`, `surface:"<surface_id>"`) using that manifest's `started`/`ended` as the two timestamps. Doing this driver-side after join is what keeps the log append race-free (sub-agents never write it). A `failed` manifest with no timestamps is skipped (no span).
5. On a per-surface failure: retry that one sub-agent once; if it fails again, carry the `failed` forward — step-06's bounded-retry / `RF-12` path handles terminal failure (do not silently drop a surface).

Proceed to `step-05-compose-route.md` with the collected manifests.
