3. Variant metadata

Side-by-side prose comparison — what each variant is built for, what it does well, what it gives up, and when to pick it.

CAREFUL-DEFAULT

Built for

Importer

Design philosophy

Standard-density single-column upload with accuracy-leaning confirmation gates and a feedback-rich error inspection list.

Strengths

  • Validation status is visible at every step of the upload
  • Explicit save bar prevents accidental submission
  • Error list shows row, column, rule, and offending value

Weaknesses

  • Single-column form takes more vertical scroll for many fields
  • Per-row confirm slows experienced operators on retry
  • Verbose banners add height that experts may want to suppress

Trade-off

Slower per upload but harder to commit a bad file; suits importers who would rather catch errors than rush.

Use when

First-time or low-frequency importers who care more about getting it right than about throughput.

POWER-DENSE

Built for

Importer

Design philosophy

Dense multi-column upload with keyboard-first inline-edit on the validation-error grid; built for the daily high-volume Importer.

Strengths

  • All 14 validation errors visible on one screen, no scrolling
  • Fix errors inline without leaving the table
  • Keyboard shortcuts cover every action

Weaknesses

  • Eight-column error grid is intimidating for first-time users
  • Hidden shortcuts must be learned before they pay off
  • Dense form layout leaves little room for help text

Trade-off

Faster for daily importers; harder to absorb on first contact.

Use when

Daily importers who handle many batches and want to fix errors inline at speed.

States per screen

S-02 — Upload accepted
default | filelog-ready
S-03 — Upload failed — validation errors
default | error-row | editing | saved | empty-after-fix
