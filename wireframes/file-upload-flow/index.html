3. Variant metadata

Side-by-side prose comparison — what each variant is built for, what it does well, what it gives up, and when to pick it.

CAREFUL-DEFAULT-IMPORTER

Built for

Importer

Design philosophy

Scrollable single-form upload with explicit save bar; default-density file log with per-row confirm on destructive actions; help text on non-obvious fields. Suits the careful daily Importer who values explicit save points and visible recovery paths.

Strengths

  • Explicit save bar on S-01 makes the commit point unambiguous; reduces accidental-submit risk for the daily Importer reviewing metadata before upload.
  • Per-row modal-confirmation on destructive Cancel actions (S-02) honours GR-04 with default focus on Keep; bulk-select stays disabled so a slip cannot cascade.
  • Section-grouped form (file / metadata / progress) keeps the cognitive load proportional to the step the Importer is on.
  • Help text on non-obvious fields (FileSetting, business date) lowers the cost of an Importer returning after time away from the screen.
  • Standard-density table on S-02 stays scannable at 20 rows per page (GR-11) without requiring keyboard-shortcut training.

Weaknesses

  • Throughput is lower than the POWER-DENSITY-EXPERT variant: explicit Upload click and per-row confirms add seconds per file for a daily operator at high volume.
  • Save bar plus separate progress section adds vertical scroll on S-01; small viewports may not see both submit CTA and validation banner at once.
  • No bulk actions on S-02 means cancelling several rejected files requires N modal traversals; intentional cost of the careful posture.
  • Density-0 table caps the at-a-glance column budget below what a daily Importer could comfortably scan if columns were tighter.
  • No keyboard shortcuts surfaced visually; an experienced Importer who later wants speed has no obvious upgrade path inside this variant.

Trade-off

Careful-first; lower throughput. Optimises for explicit save points and reversibility over per-task speed.

Use when

Daily Importer who values explicit save / undo recovery paths over keystroke efficiency; new-to-product or risk-averse operators; teams where a mis-uploaded batch has high downstream blast radius.

POWER-DENSITY-EXPERT

Built for

Importer

Design philosophy

Dense compact file-log table optimised for keyboard-first daily Importer; submit-on-Enter upload form; minimal confirmation friction on row actions. Maximises throughput for the daily high-volume operator.

Strengths

  • S-02 file-log table renders 9 columns in one viewport without horizontal scroll on a 1280px desk-class screen (D3+2 → table.compact + bulk-select column).
  • Submit-on-Enter from any field in S-01 lets the Importer drop a file, tab through metadata, and ship without touching the mouse (D1+1 → single-form.compact).
  • Keyboard shortcuts on S-02 (j/k row nav, Space select, r retry, d download, x cancel) collapse a 4-click mouse path to one keystroke (D1+1 + D2+1).
  • Inline-edit on S-02 Priority column commits-on-blur — no separate edit modal, no second confirmation gate for non-destructive field changes.
  • S-03 dashboard exposes F-15 audit columns as 4 KPI tiles, so the Importer never has to drill to find who uploaded what — daily audit-by-glance is free.

Weaknesses

  • S-02 with 9 visible columns is cognitively dense — first-time users will spend their first session orienting against the column meaning rather than transacting.
  • Tooltip-on-hover is the only chrome surfacing the keyboard shortcuts; an Importer who never hovers over the help text will not discover j/k/r/d/x — discoverability is poor by design (D2+1).
  • S-04 retry-validation requires the user to read 8 reason codes per row and decide whether a retry is appropriate without inline guidance — assumes BR rule familiarity.
  • The bulk-retry pattern lets the user select rows that are unfit for retry (e.g. genuinely unknown counterparty) and trigger a sure-to-fail retry batch — the cost of D1+1 + D2+1 is occasional wasted retry cycles.
  • S-01's submit-on-Enter posture means a stray Enter mid-edit can ship the upload before the user is fully ready — speed bias trades against accuracy.

Trade-off

Expert-first; low learnability. Maximises throughput for daily Importers who have used a similar product before; punishes occasional users with sparse on-screen guidance.

Use when

Bind to a daily, high-volume Importer who has done ≥1 week of supervised onboarding or has used a comparable file-ingestion product before. Not a good fit for first-time users or for personas who only touch this surface monthly.
