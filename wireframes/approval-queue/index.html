3. Variant metadata

Side-by-side prose comparison — what each variant is built for, what it does well, what it gives up, and when to pick it.

SPACIOUS-DEFAULT

Built for

Approver

UX posture

Error-Averse / High-Stakes

Design philosophy

Spacious deliberate review with modal confirm gates; for the Approver's high-stakes decisions.

Strengths

  • Calm, spacious layout reduces misreading high-value records
  • Reject reason captured before the decision is final
  • Confirmation step guards against accidental approvals

Weaknesses

  • Slower per decision than a dense, scan-heavy layout
  • More clicks for reviewers handling large queues
  • Extra whitespace means less fits on screen at once

Trade-off

Trades reviewing speed for fewer mistakes on irreversible, high-stakes decisions.

Use when

Approvers making final, irreversible decisions where a wrong call is costly.

DENSE-LEAN

Built for

Importer

UX posture

Analytical / Information-Dense

Design philosophy

Dense scannable table with inline row-expand detail; for the Importer's high-volume review.

Strengths

  • Many transactions visible at once for fast scanning
  • Detail opens inline without leaving the queue
  • Compact rows keep more of the batch in view

Weaknesses

  • Tight rows are harder to read carefully
  • Less room for status and amount to stand out
  • Dense layout can feel busy on small screens

Trade-off

Shows far more of the batch at once, but trades the calm spacing that makes careful single-record review easier.

Use when

High-volume reviewers scanning many imported transactions per file.
