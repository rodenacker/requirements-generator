File upload form

Primary content

Importer-only screen. Only the Importer role can upload transaction files. Approvers do not see this screen.

Fields marked with * are required. Validation happens when you move off a field; the system will not re-validate on every keystroke.

Inactive file settings appear greyed out and cannot be selected.

Auto-fills when you pick a file setting above. Confirms you selected the right one.

CSV format only. Maximum file size is set per file setting.

State — validating after submit

Rendered while the system runs server-side checks on the uploaded file. The form is disabled; the user waits for the result.

Validating your file… This usually takes a few seconds for files under a thousand rows. You will be returned to the File Log list when validation finishes.

State — error: invalid format

Rendered when the file fails an upload-time check (wrong format, empty file, too large). Per-field errors point at exactly which field is wrong; a banner summarises the failure.

Upload failed. The file you chose is not a valid CSV. Choose a different file and try again.

CSV files only. The file you selected is not a CSV.

State — success

Rendered when validation passes. The system returns the Importer to the File Log list where the new file log appears at the top of the table.

File uploaded. Your file is being processed. You will be returned to the File Log list in a moment.

Confirmation — cancel before upload

Rendered when the user clicks Cancel after filling part of the form. Default focus is on Keep — accidental Enter-presses do not discard the in-progress upload.

Discard this upload?

You have selected a file setting and a file but haven't uploaded yet. Cancelling now will discard your selections.
