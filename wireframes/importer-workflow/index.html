3. Variant metadata

Side-by-side prose comparison — what each variant is built for, what it does well, what it gives up, and when to pick it.

CAREFUL-DEFAULT

Built for

Importer

Design philosophy

Spacious layout with confirmation gates; protects daily Importers from wrong-file uploads.

Strengths

  • Generous whitespace lets daily Importers spot the right file before clicking.
  • Every destructive action opens a confirm step with default focus on Keep.
  • Required fields and inline help reduce wrong-file uploads on the upload form.

Weaknesses

  • Fewer rows fit on screen, so high-volume reviewers will scroll more often.
  • Extra confirmation steps slow the keyboard-first experts who never miss-click.
  • Spacious layout means less context visible side-by-side for deep audits.

Trade-off

Trades throughput for safety: Importers move slower but make fewer expensive mistakes on file uploads and cancellations.

Use when

Importers who upload daily but care more about getting it right than getting it done fast.

POWER-DENSE

Built for

Importer

Design philosophy

Dense table with keyboard-first inline actions; daily Importers scan and act fast.

Strengths

  • Many records visible at once with eight columns and tight rows
  • Keyboard shortcuts on common row actions; submit from any field with Enter
  • Bulk-select speeds repeat actions across many files at once

Weaknesses

  • Dense tables can overwhelm first-time or low-frequency users
  • Keyboard hints are subtle; new users may miss the fast paths
  • Tight rows leave less breathing room when reviewing errors

Trade-off

Faster scanning and action for daily importers; less friendly for occasional users who need more space and explicit guidance.

Use when

Daily importers handling many files who know the product and prefer keyboard speed.
